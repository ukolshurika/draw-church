# Geo-Name Normalization

Нормализация названий населённых пунктов Российской Империи (XIX век) из
метрических книг Серпуховского, Алексинского, Тарусского и смежных уездов.

## Зачем

Сырые данные поступают из API Яндекс.Архива. Одно и то же поселение
встречается в десятках вариантов:

| Вариант (raw)                                          | Каноническое       |
|--------------------------------------------------------|--------------------|
| `Серпуховский уезд, Пущинская волость, деревня Гавшино` | `Деревня Гавшино`  |
| `деревня Гавшина`                                      | `Деревня Гавшино`  |
| `сельцо Глазечка`                                      | `Сельцо Глазечня`  |
| `та же деревня`                                        | *(разрешается из контекста)* |
| `Серпуховской уезд`                                    | `Серпуховский уезд` |

Без нормализации:
- Дедупликация персон (`parse.py`) считает жителей одного села разными людьми
- Граф связей распадается на сотни мелких компонент вместо одного связного

## Инструменты

| Файл                      | Назначение                                      |
|---------------------------|-------------------------------------------------|
| `normalize_geo.py`        | Массовая нормализация `all-nodes.json`          |
| `settlement-synonyms.json` | Словарь синонимов (617+ записей)                |
| `reader.py`               | Рантайм-нормализация: `normalize_settlement()`  |

### Область покрытия и расширение

Текущий `settlement-synonyms.json` (617 записей) покрывает **Богоявленский
приход г. Серпухова** и окрестные волости (Пущинская, Высотская, Туровская,
Кошкинская). Скрипты `scraper.py` / `parse.py` / `normalize_geo.py` не
привязаны к конкретному приходу и работают с любыми метрическими книгами
Яндекс.Архива — но при смене прихода словарь синонимов **необходимо
расширять**.

При добавлении данных из нового прихода:

1. Запустить `parse.py` → получить `all-nodes.json`
2. Запустить `normalize_geo.py` — он выведет список уникальных поселений
3. Вручную просмотреть топ-50, выявить неузнанные варианты названий
4. Добавить их в `MANUAL_SYNONYMS` (в `normalize_geo.py`) или напрямую
   в `settlement-synonyms.json`
5. Перезапустить `normalize_geo.py`

Типовые проблемы нового прихода: местные названия деревень, уникальные
написания уездов/волостей, контекстные ссылки в начале страниц без
предыдущего контекста.

## Запуск

```bash
# 1. Спарсить сырые данные (если ещё не)
python3 parse.py

# 2. Нормализовать поселения
python3 normalize_geo.py

# 3. Перестроить визуализацию
python3 viz.py
```

## Алгоритм нормализации

### Шаг 1 — Извлечение ядра

Строка разбивается по запятым/точкам. Для каждой части ищется шаблон
`{тип} {имя}`, где тип — `деревня|село|сельцо|город|слобода|погост|хутор|посад|завод`.

Приоритет — последняя часть с именем поселения. Административный контекст
(уезд, волость, губерния) отбрасывается.

```
"Серпуховский уезд, Пущинская волость, деревня Гавшино"
  → части: ["Серпуховский уезд", "Пущинская волость", "деревня Гавшино"]
  → ядро: "деревня Гавшино"
```

### Шаг 2 — Грамматическая нормализация

Имя поселения приводится к именительному падежу через словарь синонимов.
Типовые паттерны:

- **Родительный падеж**: `Гавшина → Гавшино`, `Лопатина → Лопатино`
- **Мн. число → ед. число**: `Гавшины → Гавшино`, `Дашковки → Дашково`
- **Опечатки/фонетика**: `Тверизино → Тверитино`, `Глазечна → Глазечня`

### Шаг 3 — Нормализация административных единиц

Названия уездов и волостей тоже страдают от вариативности:

```
Серпуховской уезд → Серпуховский уезд
Тарузский уезд    → Тарусский уезд
Испенская волость → Испанская волость
Кущинская волость → Кошкинская волость
```

### Шаг 4 — Разрешение контекстных ссылок

Выполняется в `parse.py`. Строки вида `та же деревня`, `то же сельцо`
разрешаются из контекста предыдущих записей (внутри страницы и между ними).

## Добавление новых синонимов

### Через словарь settlement-synonyms.json

```json
{
  "новый_вариант": "Каноническое_имя"
}
```

Ключ — lowercase-форма варианта, значение — каноническое имя с заглавной буквы.

### Через reader.py (рантайм)

```python
from reader import add_settlement_synonym
add_settlement_synonym("очепятка", "Правильно")
```

### Массово — в normalize_geo.py

Добавить в словарь `MANUAL_SYNONYMS`. Скрипт также авто-генерирует
составные записи (`{слово} уезд`, `{слово} волость`) для административных
единиц.

## Контекстные ссылки

`parse.py` разрешает:

| Сырая строка                  | Механизм разрешения                |
|-------------------------------|------------------------------------|
| `та же деревня`               | Последняя `деревня` в записи/контексте |
| `то же сельцо`                | Последнее `сельцо` в записи/контексте  |
| `то же село`                  | Последнее `село` в записи/контексте    |
| `тот же уезд, деревня N`      | Подстановка последнего уезда          |
| `той же помещицы`             | Подстановка последнего помещика       |

Разрешение происходит **внутри записи** (entry) и **между записями** страницы
через `global_ctx`, передаваемый между вызовами `process_entry()`.

## Типы поселений

Используемые в метриках Серпуховского/Алексинского уездов:

| Тип       | Значение                              |
|-----------|---------------------------------------|
| `деревня` | Крестьянское поселение без церкви     |
| `сельцо`  | Помещичье поселение без церкви        |
| `село`    | Поселение с церковью                  |
| `город`   | Город (Серпухов, Таруса)              |
| `слобода` | Слобода (Владычная, Гостиная)         |

Тип поселения **не нормализуется** между собой — `Деревня Гавшино` и
`Село Гавшино` могут быть разными местами.
